# PRD: 한국 공시 투자 판단 에이전트

**버전:** 1.0  
**작성일:** 2026-04-11  
**상태:** 확정 (Seed 결정화 완료 · 모호성 점수 0.18)

---

## 1. 개요 (Overview)

### 1.1 배경 및 문제 정의

개인 투자자가 한국 상장사 공시를 읽을 때 다음과 같은 어려움이 반복된다.

1. **해석 비용**: 법률·회계 용어와 장문 본문으로 핵심이 무엇인지 파악하기 어렵다.
2. **맥락 부족**: 동일·유사 유형의 과거 사례와 시장 반응을 스스로 조사하기엔 시간이 많이 든다.
3. **행동 정리 어려움**: “긍정/부정”만으로는 포지션 조정(손절·익절·진입·관망)을 정리하기 어렵다.

### 1.2 프로젝트 목표

DART 원문과 KRX 시세를 연계한 **Supervisor + ReAct 멀티에이전트 파이프라인**으로, 사용자가 종목명 또는 공시 URL을 입력하면 **유형 분류 → 명세 추출 → LLM 종합 판단 → 유사 공시(peer) 탐색 → 장기 구간 포함 주가 패턴 분석**을 거쳐, **요약·근거·시나리오·네 가지 액션 라벨**이 담긴 리포트를 제공한다. MVP에서는 **기능 1(공시 투자 판단)** 완성도를 최우선으로 한다.

### 1.3 핵심 가치 제안

| 문제 | 해결책 |
|------|--------|
| 공시가 무슨 뜻인지 모르겠음 | 룰 기반 유형 분류 + 유형별 명세 추출 + 쉬운 말 요약 |
| 이게 주가에 얼마나 큰 일인지 감이 안 옴 | LLM 기반 영향도(HIGH/MEDIUM/LOW)와 근거 서술 |
| 비슷한 적이 있었는지 모름 | 동업종·동일 유형·3년 엄격 매칭 후 완화 루프로 peer 탐색 |
| 사후 주가만 감으로 본다 | 공시 전후 1주~1년 다기간 수익률 패턴을 리포트에 반영 |
| 다음에 뭘 해야 할지 정리가 안 됨 | 손절 / 익절 / 진입 / 관망 라벨과 시나리오 병행 |

## 2. 결과 — 그래서 이걸 쓰면 뭐가 좋은데?

이 제품을 쓰면 사용자는 **공시 한 건을 “읽은 것”에서 “판단 근거가 정리된 상태”로 바꿀 수 있다.**

- **시간 절감**: DART 원문 수집, 유형 판별, 핵심 수치 정리, 유사 공시 검색, 과거 주가 구간 계산을 사람이 일일이 하던 작업을 에이전트가 순차 처리한다.
- **놓치기 쉬운 리스크 가시화**: 유상증자·CB/BW·잠정실적 등 유형별로 정의된 명세(`disclosure_spec`)와 LLM 해석이 결합되어, 희석·리픽싱·일회성 이익 등 **숫자 뒤의 의미**를 문장으로 정리한다.
- **과거 맥락**: peer 기업의 공시 후 주가 패턴(단기뿐 아니라 **3개월~1년** 구간)이 리포트에 들어가, “비슷한 일이 있었을 때 시장이 어떻게 움직였는지”에 대한 **참고 프레임**을 얻는다.
- **행동 정리**: 면책·비투자권유 문구를 법적 요건에 맞게 두되, 과도한 반복 없이 **손절·익절·진입·관망** 중심으로 다음 스텝을 정리해 볼 수 있다.
- **불필요한 분석 낭비 감소**: 정기공시 등 영향이 낮은 경우(LOW)나 정정·기재정정 등 회색지대는 **조기 종료**와 짧은 안내로 혼란을 줄인다.

**한 줄 요약**: 공시를 입력하면 **“무슨 일인지 → 왜 중요한지(또는 아닌지) → 비슷했던 적은 → 그때 주가는 → 지금 관점에선 어떤 행동 라벨이 상응하는지”**가 한 번에 정리된다.

## 3. 사용 대상 (Target Users)

**주요 사용자**: 공시를 읽어도 해석·맥락 정리가 어려운 **개인 투자자**.

**사용자 특성**:

- 특정 종목 공시가 났을 때 빠르게 의미를 파악하고 싶음
- DART·KRX를 직접 오가며 조사하는 데 부담을 느낌
- 투자 결정은 본인이 하되, **근거 정리**를 도구에 맡기고 싶음

## 4. 사용자 시나리오 (User Journey)

### 기능 1 — 공시 투자 판단 (MVP 우선)

```
[1] 입력
    사용자 → 종목명 또는 공시 URL 입력

[2] Step 1 — 공시 해석 에이전트
    DART 원문 수집 → 룰 기반 유형 분류 → 명세 추출 → LLM 종합 판단
    → 영향도 HIGH / MEDIUM / LOW

[3] 분기
    LOW 또는 비대상·회색지대 → 조기 종료 + 짧은 교육/안내
    MEDIUM → peer·주가는 경량 요약 후 Step 4 중심
    HIGH → Step 2~4 전체

[4] Step 2 — 유사 공시 탐색
    동업종 + 동일 공시 유형 + 최근 3년 엄격 검색 → 부족 시 조건 완화 루프

[5] Step 3 — 주가 패턴 분석
    KRX 일별 시세 기반, 공시 전후 1주·1개월·3개월·6개월·1년 수익률

[6] Step 4 — 리포트 작성
    요약, 영향 근거, peer·주가 요약, 시나리오, 액션 라벨 4종
```

### 기능 2 — 재무 데이터 챗봇 (동일 앱 탭, 1차 완성도는 기능 1 이후)

```
사용자 → 자연어 질의
Query 에이전트 → DART 재무 API 자율 호출 → 표 + 차트 출력
```

## 5. 에이전트 아키텍처 (Agent Architecture)

### 5.1 기능 1 구조

```
Supervisor
│
├── Step 1. ReAct — 공시 해석
│   ├── dart_disclosure_fetch
│   ├── 유형 분류 (룰): 유상증자, CB/BW, 잠정실적, 합병/분할,
│   │   최대주주변경, 대규모계약, 관리종목/상장폐지
│   ├── 해당 없음 → 단순 정기공시 등 조기 종료
│   ├── 명세 추출 (disclosure_spec, null 허용, 본문 보완)
│   └── LLM 종합 판단 → HIGH / MEDIUM / LOW
│
├── Step 2. ReAct — 유사 공시 (HIGH 전체, MEDIUM은 경량)
├── Step 3. ReAct — 주가 패턴 (HIGH 전체, MEDIUM은 경량)
└── Step 4. Writer — 투자 판단 리포트
```

### 5.2 영향도별 후속 처리

| 영향도 | 조건 요약 | 후속 |
|--------|-----------|------|
| HIGH | 강한 긍·부정 + 수치 근거 비교적 명확 | Step 2~4 전체 |
| MEDIUM | 중립·애매 | Step 4 중심, peer·주가 경량 |
| LOW | 비대상 유형 또는 영향 미미 | 조기 종료, 짧은 교육 문구 |

## 6. 기능 명세 (Feature Specifications)

### 6.1 공시 입력·분류

- **입력**: 종목명, 공시 URL 등 (`disclosure_input`)
- **출력**: `disclosure_type`, `extracted_fields`, `llm_stance`, `impact_level`
- **회색지대**: 정정공시·기재정정 등 제목/본문만으로 애매하면 분석 제외 또는 조기 종료

### 6.2 Peer 탐색

- 최초: 동업종 + 동일 공시 유형 + 최근 3년
- 유사도 부족 시 검색 조건 완화 루프

### 6.3 주가 패턴

- KRX(또는 동등 일별 시세) 실데이터
- **장기 구간(3개월~1년) 반영** 필수

### 6.4 컨센서스·웹 보완

- 잠정실적 등: 웹 검색으로 컨센서스 보완, 실패 시 필드 **「미확인」** — 파이프라인은 중단하지 않음

### 6.5 리포트

- 요약, 영향 근거, 시나리오, **손절 / 익절 / 진입 / 관망** 라벨
- 면책·비권유는 법적 요건에 맞게, 과도한 반복 없이 배치

### 6.6 UI

- **Streamlit**, 탭 분리: **공시 판단** / **데이터 챗봇**

| 지표 | 정의 | 목표(방향) |

## 10. 제약사항 및 비목표 (Constraints & Non-Goals)

### 제약사항

- MVP에서 **기능 1 우선**; 기능 2는 동일 앱 탭으로 존재하나 1차 완성도는 기능 1에 둔다.
- **사용자 포트폴리오·계좌·보유 종목 연동 없음.**
- LLM은 **gpt-5-mini** 고정; 미지원 API 파라미터 사용 금지.
- 복수 유형 혼재 시 단일 LLM 입력에 모든 맥락 포함; 내부 우선순위는 보수적(주주 불리·변동성 큰 쪽) 또는 구현 시 명시 규칙.

### 비목표 (Non-Goals)

- 실제 주문·자동매매 연동
- 개인별 맞춤 포트폴리오 최적화
- 법적 투자자문을 대체한다는 표현(항상 정보 제공·의사결정은 사용자)

## 12. 수용 기준 (Acceptance Criteria) 요약

- 샘플 공시 다수에 대해 Supervisor 파이프라인 **끝까지 실행 가능**(담당자 수동 품질 확인).
- Peer: 엄격 조건 후 **완화 루프** 동작.
- 주가: **3개월~1년** 구간이 리포트에 반영.
- 컨센서스 없을 때 **「미확인」**, 파이프라인 비중단.
- LOW 시 **짧은 교육형** 설명 출력.
- 회색지대 공시는 **분석 제외·조기 종료** 경로 명확.
